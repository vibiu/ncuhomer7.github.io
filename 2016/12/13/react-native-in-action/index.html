React Native in Action

发表于 2016-12-13 |

起因

写了一年的后端, 是时候试一下移动端的水了, 也有一直在看Java, 打算用原生Java入手Android, 但是由于一直在写动态脚本语言, Java使用得还是不太熟练, 就考虑先用JavaScript现在很火的移动框架React Native写喽.

环境搭建

Android的环境搭建有点小复杂, 而且由于目前手上的电脑配置比较低, 使用Android Studio怕会比较卡, 就打算直接配置JDK和SDK.

JDK

先从Oracle官网下载相应的JDK的压缩包,解压

$ tar -xf jdk-8u101-linux-x64.tar.gz

将解压后的目录移动到安装目录, 比如~/opt, 将环境变量配置写入.bashrc或.zshrc

export JAVA_HOME=~/opt/jdk1.8.0_101
export PATH=$JAVA_HOME/bin:$PATH
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar

重开终端,这时候可以试下

$ javac --version

SDK

先从Google官网下载相应SDK压缩包, 这里我不打算使用Android Studio, 所以我只下载了 command line tools, 下好之后解压, 并把解压出的目录移动到安装目录, 比如~/opt, 将环境变量配置写入.bashrc或.zshrc

export ANDROID_HOME=~/opt/android-sdk-linux
export PATH=$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools:$PATH
Node.js

这里使用nvm这个node多版本管理器安装器.
这篇教程相当详细, 值得参考.

git clone https://github.com/cnpm/nvm.git ~/.nvm && cd ~/.nvm && git checkout `git describe --abbrev=0 --tags`

把nvm目录移动到安装目录, 比如~/opt, 在.bashrc 或.zshrc中写入

source ~/opt/nvm/nvm.sh

重开终端nvm应该已经有用

$ nvm --version

使用nvm安装任意版本的node都相当简单, 比如现在最新版的6.8.0

$ nvm install 6.8.0
$ nvm use 6.8.0

现在版本为6.8.0的node已经安装好,可以把它设为默认的版本

$ nvm alias default 6.8.0

React-native-cli

react有比较详细的指导
使用node的包管理器npm安装react-native-cli

$ npm install -g react-native-cli

使用react-native-cli生成应用目录

$ react-native init AwesomeProject
$ cd AwesomeProject
$ react-native run-android

Troubleshooting

如果只是按照上面的步骤, 会发现运行失败, 部分问题可以在React的 Troubleshooting page找到解决方法
而更多的是遇到Android的问题, 比如

Failed to find Build Tools revision 23.0.1

这是因为新安装的SDK版本比较新, 但是react要求build tools版本必须是23.0.1.
Stack Overflow上有解决方法

$ android list sdk -a
$ $android update sdk -a -u -t 3

使Android设备与电脑连接一般是自动完成的, 但可能 no permission, 同样可以在Stack Overflow上找到解决方法

--> /etc/udev/rules.d/51-android.rules
    SUBSYSTEM=="usb", ATTR{idVendor}=="04E8", MODE="0666", GROUP="plugdev"

$ adb kill-server
$ adb start-server
$ adb devices

注意Android设备需要打开调试模式, 并使连接类型改成照相模式.
而下面这个错误就比较蛋疼

“aapt” IOException error=2, No such file or directory"

这是因为Aapt is a 32bit application. 在64bit的环境下无法正常运行, 同样在Stack OverFlow上能找到解决方法
这在各种操作系统上有所不同, 比如在Arch Linux上需要安装Multilib

$ sudo pacman -Syv lib32-libstdc++5 lib32-zlib

差不多以上就是遇到的各种坑, 如果你不幸遇到其它问题, Google it, 或者在Statck OverFlow上找找, 总可以找到解决方法.

Developing

基本现在react-native run android可以正常运行了, 会在Android设备上自动安装你的应用, 可以通过摇动打开DEBUG模式.
React的模板思想和现在的Vue的模板还有一点差距, 后者专注于数据的渲染, 性能可观.
比较欣喜的是React默认支持fetch进行网络通信, 也支持ES2017.
推荐的Sublime-text插件Babel, 它能为JSX提供语法高亮.

The end

Having fun in React Native.
